The EZ
Language

A dynamically typed, tree-walk interpreted language built in C++17. Reads like plain English, ships with a web server, SQLite, an HTTP client, async threads, and a package manager — all out of the box.

Dynamically Typed C++17 Interpreter Full OOP Built-in SQLite Built-in Web Server Package Manager First-class Functions Async / spawn
ez · quickstart
// A quick taste of EZ model Person { init(name, age) { self.name = name self.age = age } task greet() { out `Hi, I'm {self.name} — age {self.age}` } } people = [new Person("Alice", 30), new Person("Bob", 17)] get p in people { p.greet() when p.age >= 18 { out ` ↳ adult` } } // Higher-order + lambda adults = filter(people, (p) => p.age >= 18) names = map(adults, (p) => p.name) out join(names, ", ") // SQLite in 3 lines db = dbOpen("app.db") dbExec(db, "CREATE TABLE IF NOT EXISTS log (msg TEXT)") dbExec(db, "INSERT INTO log VALUES (?)", ["started"])
Plain-English syntax
when = if, task = function, give = return, model = class. Code reads like prose.
Batteries included
SQLite, HTTP server, libcurl HTTP client, JSON, async threads, file I/O — no imports needed, all built into the runtime.
Full OOP
Classes (model), constructors (init), single inheritance (extends), interfaces (implements), hidden/shown, static, super.
First-class functions
Functions are values. Arrow lambdas, pipe lambdas, closures, default parameters, higher-order built-ins like map/filter/reduce.
Two comment styles
// line comments, # hash comments, and /* nested block comments */.
Pipeline: Lexer → Parser → Interpreter
Classic tree-walk interpreter. Lexer tokenises → Parser builds AST → Interpreter walks it. Errors include line numbers.
01
Getting Started

Installation

EZ is built from source. You need a C++17 compiler and a handful of libraries on Windows.

Prerequisites

  • C++17 compiler — GCC (MinGW-w64), Clang, or MSVC 2019+
  • SQLite3 — development headers + sqlite3.lib
  • libcurl — development headers + lib (for HTTP client and package manager)
  • WinSock2 — bundled in the Windows SDK (ws2_32.lib)

Clone & compile

$git clone https://github.com/imabd645/EZ-language.git && cd EZ-language
bash
g++ -std=c++17 -O2 -Wall -o ez.exe \ src/main.cpp src/Lexer.cpp src/Parser.cpp \ src/Interpreter.cpp src/Builtins.cpp \ -I src -lws2_32 -lsqlite3 -lcurl

Add to PATH

Create C:\ezlang\, copy ez.exe and all required .dll files there, then add the folder to your Windows PATH environment variable. Restart your terminal.

CLI reference

CommandDescription
ezLaunch the interactive REPL
ez <file.ez>Run an EZ script file — exits 65 on lexer/parser error
ez install <pkg>Install a package from GitHub
ez install <pkg> <branch>Install a specific branch / version
ez listList all installed packages in C:/ezlib
ez init <name>Scaffold a new package with package.ez manifest
ez --helpPrint usage summary

REPL

Running ez with no arguments launches an interactive read-eval-print loop. The REPL detects unclosed { } braces and waits for the block to complete before executing, shown by the ... prompt.

repl session
EZ Language Interpreter v1.0 Type 'exit' to quit >>> x = 42 >>> out x 42 >>> task add(a, b) { ... give a + b ... } >>> out add(10, 5) 15 >>> exit Goodbye!

Core Language

Data Types

EZ is dynamically typed — every value carries its type at runtime. There are nine value types, all represented by the Value variant type internally.

number
42, 3.14, -7
64-bit IEEE-754 float. No integer type — all numbers are doubles.
string
"hi", 'hi', `hi`
Immutable text. Three literal forms: double, single, and backtick template strings.
bool
true / yes
Boolean. yes/no are aliases for true/false.
nil
nil
Absence of value. Functions without give return nil.
array
[1, "a", true]
Ordered, mutable, mixed-type. Zero-indexed. Shared via pointer.
dictionary
{key: val}
Hash map. Keys are always strings. Dot or bracket access.
function
task foo(){}
First-class. Captures its environment (closure).
instance
new Dog("Rex")
Object created from a model. Holds properties and bound methods.
future
spawn(fn)
Handle to an async computation. Resolve with await().

Truthiness rules

Only two values are falsy: nil and false. Everything else — including 0, "", and [] — is truthy. This differs from many languages.

ez
when 0 { out "truthy" } // ← runs! 0 is truthy in EZ when "" { out "truthy" } // ← runs! when [] { out "truthy" } // ← runs! when nil { out "x" } // never runs when false{ out "x" } // never runs

Equality semantics

== compares values deeply: two arrays are equal if they have the same length and all elements are equal. Two instances are only equal if they are the exact same object (reference equality). Two functions are never equal to each other.

Runtime type inspection

ez
out type(42) // "number" out type("hi") // "string" out type(true) // "bool" out type(nil) // "nil" out type([1,2]) // "array" out type({a:1}) // "dictionary" out type(new Dog("")) // "instance" out type(type) // "function" (native)
⚠
No integer type

All numbers are 64-bit floats. 10 / 3 → 3.333…. Use floor(), ceil(), or round() when you need whole numbers.

Core Language

Variables & Operators

Variables are created by assignment — no declaration keyword. Scope is lexical: each { } block creates a new child scope.

Assignment

ez
x = 10 name = "Alice" active = true nothing= nil scores = [90, 85, 92] config = { debug: false, port: 8080 } // Compound assignment operators x += 5 // x = x + 5 x -= 2 x *= 3 x /= 4
ℹ
Scope walk on write

On assignment EZ walks up the scope chain looking for an existing binding. If it finds one it updates it (allowing closures to mutate outer variables). If nothing is found anywhere the variable is created in the current scope.

Core Language

Strings

Strings are immutable sequences of characters. Three literal forms are available; the backtick form supports full expression interpolation.

ez · all string forms
// Double quotes s1 = "Hello, World!" // Single quotes — identical behaviour s2 = 'Hello, World!' // Backtick template strings — interpolate any expression with { } name = "Alice" age = 30 items = ["a", "b"] s3 = `Hello, {name}!` // Hello, Alice! s4 = `She is {age} years old` // She is 30 years old s5 = `List has {len(items)} items` // List has 2 items s6 = `2 + 2 = {2 + 2}` // 2 + 2 = 4 s7 = `UPPER: {upper(name)}` // UPPER: ALICE // Concatenation with + full = "Hello, " + name + "!" // String repetition with * line = "-" * 30 // "------------------------------" // Character-by-character iteration get ch in "hello" { out ch } // h e l l o // Index access — returns single-char string out s1[0] // "H"

Escape sequences

SequenceCharacter
\nNewline (LF)
\tTab
\rCarriage return
\\Literal backslash
\"Double quote inside "…"
\'Single quote inside '…'
\`Backtick inside `…`
\{Literal { inside template strings
\}Literal } inside template strings

How template strings work internally

When the lexer encounters a backtick string it expands it at tokenisation time. `Hello {name}!` becomes the token stream equivalent of ("Hello " + str(name) + "!"). This means the full power of EZ expressions — function calls, arithmetic, method calls — is available inside the { } braces.

06
Core Language

Comments

Three comment forms. Block comments can be nested.

ez
// Double-slash line comment x = 10 // inline comment # Hash line comment — alternative style y = 20 # also a comment /* Block comment — spans multiple lines. /* Nesting is supported — the lexer tracks depth. This inner comment is valid. */ Back to the outer block. */ out x + y
✦
Nested block comments

EZ's lexer tracks nesting depth for /* */, so you can safely comment out blocks that already contain block comments — a common need during development. A mismatched closing tag emits an "Unterminated block comment" error.

Control Flow

when / other

Conditional execution. when is EZ's if; other is else. Chain with other when for else-if.

ez
// Basic when when x > 10 { out "big" } // when / other when age >= 18 { out "adult" } other { out "minor" } // Chained: other when = else-if when score >= 90 { grade = "A" } other when score >= 80 { grade = "B" } other when score >= 70 { grade = "C" } other { grade = "F" } // Braceless single-statement form when x > 0 out "positive" other out "non-positive" // Logical operators in condition when x > 0 and x < 100 { out "in range" } when name == "admin" or role == "superuser" { out "privileged" } when not active { out "inactive" }
08
Control Flow

while

Loop while a condition remains truthy. Condition is tested before each iteration.

ez
// Count up i = 0 while i < 5 { out i i += 1 } // Infinite loop with escape while true { answer = input("Enter 'quit' to stop: ") when answer == "quit" { escape } out `You entered: {answer}` }
09
Control Flow

repeat … to

Counted loop. Iterates a loop variable from start to end — inclusive both ends. Supports both ascending and descending iteration.

ez
// repeat varName = start to end (inclusive both ends) repeat i = 0 to 4 { out i // 0, 1, 2, 3, 4 } // Descending (start > end) repeat n = 5 to 1 { out n // 5, 4, 3, 2, 1 } // Expressions as bounds size = len(items) - 1 repeat idx = 0 to size { out items[idx] } // Nested repeat — times table repeat r = 1 to 5 { repeat c = 1 to 5 { out `{r}×{c}={r*c}` } }
⚠
Inclusive upper bound

repeat i = 0 to 4 runs for i = 0, 1, 2, 3, 4. Both bounds are inclusive. This differs from range() which is exclusive at the upper end.

10
Control Flow

get … in

Foreach loop. Works over arrays, strings (character-by-character), and dictionaries (iterates keys).

ez
// Over array fruits = ["apple", "banana", "cherry"] get fruit in fruits { out fruit } // Over range() get n in range(5) { out n } // 0,1,2,3,4 get n in range(3, 8) { out n } // 3,4,5,6,7 // Over string — yields one character per iteration get ch in "hello" { out ch } // h e l l o // Over dictionary — yields KEYS as strings config = { host: "localhost", port: 8080 } get k in config { out `{k} = {config[k]}` }
11
Control Flow

escape / skip

escape breaks out of the innermost loop. skip jumps to the next iteration. Both work inside while, repeat, and get.

ez
// escape = break repeat i = 0 to 10 { when i == 5 { escape } out i // 0 1 2 3 4 } // skip = continue repeat i = 0 to 10 { when i % 2 == 0 { skip } out i // 1 3 5 7 9 } // Search-and-escape pattern found = false get item in data { when item == target { found = true escape } }
12
Functions

task (functions)

Functions are defined with task. They are first-class values — assign them to variables, pass them as arguments, return them from other functions. Functions close over their defining environment.

ez
// Basic function task greet(name) { out `Hello, {name}!` } greet("World") // Return value with give task add(a, b) { give a + b } result = add(3, 4) // 7 // Default parameters — must come after required params task connect(host, port = 8080, ssl = false) { out `{host}:{port} ssl={ssl}` } connect("localhost") // localhost:8080 ssl=false connect("api.example.com", 443, true) // Early return task safeDivide(a, b) { when b == 0 { give nil } give a / b } // Functions as values task double(x) { give x * 2 } transform = double out transform(5) // 10 // Higher-order function task applyTwice(fn, x) { give fn(fn(x)) } out applyTwice(double, 3) // 12

Function arity & error handling

If you call a function with the wrong number of arguments (and no defaults cover the gap), the runtime throws a RuntimeError naming the expected and actual counts. Native built-in functions use arity = -1 to signal variadic.

13
Functions

Lambda & closures

EZ supports two anonymous function syntaxes. All functions — named or anonymous — close over their defining environment, enabling factory functions, counters, and memoisation.

ez · lambda forms
// Arrow form: (params) => expr double = (x) => x * 2 add = (a, b) => a + b noArgs = () => "hello" // Block body arrow process = (x) => { y = x * 2 give y + 1 } // Pipe form: |params| => expr (alternative syntax) square = |x| => x * x sumTwo = |a, b| => a + b // Pipe with block body clamp = |val, lo, hi| { when val < lo { give lo } when val > hi { give hi } give val }

Closures

ez · closures
// Counter — inner function captures outer variable task makeCounter(start = 0) { count = start give () => { count += 1 give count } } c1 = makeCounter() out c1() // 1 out c1() // 2 out c1() // 3 c2 = makeCounter(10) out c2() // 11 — independent state // Factory returning specialised function task multiplier(factor) { give (x) => x * factor } triple = multiplier(3) out triple(7) // 21

Data Structures

Arrays

Ordered, mutable, mixed-type sequences. Zero-indexed. Internally stored as a std::vector<Value> allocated on the heap and shared by pointer — mutations propagate to all references.

ez
// Create nums = [1, 2, 3, 4, 5] mixed = ["hi", 42, true, nil] nested = [[1,2], [3,4]] empty = [] // Read (0-based index) out nums[0] // 1 out nums[4] // 5 // Write nums[0] = 99 // Add / remove (mutate in-place) push(nums, 6) // append, returns array last = pop(nums) // remove + return last insert(nums, 0, 0) // prepend 0 remove(nums, 2) // remove at index 2, returns removed value // Non-mutating transforms (return new array) sorted = sort(nums) rev = reverse(nums) part = slice(nums, 1, 4) // index 1,2,3 // Query size = len(nums) found = contains(nums, 3) idx = indexOf(nums, 3) // -1 if not found // Array + array concatenation combined = [1,2] + [3,4] // [1,2,3,4] // Create with range() r1 = range(5) // [0,1,2,3,4] r2 = range(3, 8) // [3,4,5,6,7]
⚠
Out-of-bounds is a runtime error

Accessing arr[idx] where idx < 0 or idx >= len(arr) throws a RuntimeError: Array index out of bounds. Always guard with a length check or try/catch.

Data Structures

Dictionaries

Hash maps with string keys and any-type values. Use brace literal syntax; access with dot notation or bracket notation. Internally an unordered_map<string, Value>.

ez
// Create — colon separator, or equals person = { name: "Alice", age: 30, active: true } point = { x = 10, y = 20 } // equals also works as separator empty = {} // Access — dot or bracket notation out person.name // "Alice" out person["age"] // 30 // Add / modify person.email = "[email protected]" person["age"] = 31 // Missing key returns nil (no error) out person["missing"] // nil // Check existence out has_key(person, "email") // true out "name" in person // true (in operator) // Remove a key dictRemove(person, "active") // Iterate keys get k in person { out `{k}: {person[k]}` } // get all keys / values out keys(person) out values(person) // Nesting cfg = { db: { host: "localhost", port: 5432 } } out cfg.db.host // "localhost"

OOP

model (classes)

Define object types with model. Use init for the constructor, self inside methods to reference the current instance, and new ModelName(…) to instantiate. If a model defines a toString task the interpreter calls it automatically on string conversion.

ez · model full example
model BankAccount { init(owner, balance) { self.owner = owner hidden self.balance = balance // private property } shown task deposit(amount) { when amount <= 0 { throw "Amount must be positive" } self.balance += amount } shown task withdraw(amount) { when amount > self.balance { throw "Insufficient funds" } self.balance -= amount } shown task getBalance() { give self.balance } // static — belongs to class, not instance static task create(owner) { give new BankAccount(owner, 0) } // Automatic string conversion task toString() { give `BankAccount({self.owner}, ${self.balance})` } } // Static method call via class name acc = BankAccount.create("Alice") acc.deposit(1000) acc.withdraw(250) out acc.getBalance() // 750 out acc // BankAccount(Alice, $750) ← toString()

Access modifiers

ModifierMeaningUsage
shownPublic — default if omittedshown task getName() { … }
hiddenPrivate — only accessible from inside the model. A RuntimeError is thrown if accessed externally.hidden self.balance = 0
staticClass-level member — accessed via ClassName.memberstatic task create() { … }

Property declarations in the class body

Properties can be declared directly in the model body outside init. Optionally give them an initial value. Static properties with an initializer are evaluated once at class-definition time.

ez
model Config { static version = "1.0" // class-level constant hidden debug = false // private default init() {} task enableDebug() { self.debug = true } } out Config.version // "1.0"
17
OOP

Inheritance

EZ supports single inheritance via extends. Use super.init(…) to call the parent constructor and super.methodName() to call a parent method. Method lookup walks up the class hierarchy automatically.

ez
model Animal { init(name, sound) { self.name = name self.sound = sound } task speak() { out `{self.name} says: {self.sound}` } task describe() { out `I am {self.name}` } } model Dog extends Animal { init(name, breed) { super.init(name, "Woof") // call parent constructor self.breed = breed } task speak() { // override super.speak() // call parent method out `(breed: {self.breed})` } } model GuideDog extends Dog { // multi-level chain init(name) { super.init(name, "Labrador") self.trained = true } } rex = new Dog("Rex", "Labrador") buddy = new GuideDog("Buddy") rex.speak() // Rex says: Woof / (breed: Labrador) rex.describe() // inherited from Animal buddy.speak() // Buddy says: Woof / (breed: Labrador) out buddy.trained // true
18
OOP

Interfaces

Declare a set of required method names. Any model that implements an interface is checked at class-definition time — a RuntimeError is thrown immediately if any declared method is missing.

ez
interface Serializable { task serialize() task deserialize(data) } interface Drawable { task draw() task resize(factor) } // A model can implement multiple interfaces model Shape implements Drawable, Serializable { init(kind, size) { self.kind = kind self.size = size } task draw() { out `Drawing {self.kind} at size {self.size}` } task resize(f) { self.size *= f } task serialize() { give to_json({ kind: self.kind, size: self.size }) } task deserialize(data) { obj = parse_json(data) self.kind = obj["kind"] self.size = obj["size"] } }
19
OOP

Structs

Structs are lightweight named records. They compile to a model whose init sets each declared field. Use them for simple data containers without behaviour.

ez
struct Point { x y } struct Color { r g b } struct Rect { x y width height } // Instantiate — positional arguments match field order p = new Point(10, 20) c = new Color(255, 0, 128) out p.x // 10 out c.r // 255 // Fields are mutable p.x = 99
✦
struct vs model

Use struct for plain data records. Use model when you need methods, access control, inheritance, or a custom constructor. Under the hood a struct simply generates a model whose init assigns each field from positional arguments.

Error Handling

try / catch / throw

throw any value as an error — string, number, dictionary, or instance. catch receives the thrown value (or the error message string for native RuntimeErrors) in the named variable.

ez
// Basic try / catch try { result = num("not a number") } catch (err) { out `Caught: {err}` } // throw a string task divide(a, b) { when b == 0 { throw "Division by zero" } give a / b } // throw a structured dictionary — rich error objects task validateAge(age) { when age < 0 { throw { code: "INVALID", msg: "Age cannot be negative", got: age } } give age } try { validateAge(-5) } catch (e) { when type(e) == "dictionary" { out `[{e["code"]}] {e["msg"]} (got {e["got"]})` } other { out `Error: {e}` } } // Re-throw — propagate after logging try { riskyOperation() } catch (e) { appendLine("errors.log", e) throw e }
ℹ
Internals

Native runtime errors (wrong argument count, index out of bounds, undefined variable, etc.) are thrown as C++ RuntimeError exceptions and caught by EZ's catch. The caught value in EZ is always a string — the error message. When you throw a dictionary in EZ, it is stringified to its toString() value before being wrapped in a RuntimeError.

use (imports)

use loads and executes an EZ file in the current interpreter context. Everything defined in that file becomes available in the current scope.

ez
// Import a local file use utils.ez myUtil() // function defined in utils.ez // Import an installed package (looks in C:/ezlib/math/) use math out math.sqrt(25)

Module resolution order

  1. Try exact path as given (relative to working directory)
  2. Try C:/ezlib/<name> (direct file in lib folder)
  3. Read C:/ezlib/<name>/package.ez and load the main field
  4. Try C:/ezlib/<name>.ez
  5. Try C:/ezlib/<name>/main.ez
  6. Throw RuntimeError: Could not find module '<name>'

Advanced

Runtime internals

Understanding how EZ works under the hood — useful when debugging or optimising programs.

Execution pipeline

  1. Lexer — tokenises source into Token structs with type, lexeme, literal value, line, and column. Reports lexer errors to stderr and sets hadError = true.
  2. Parser — builds an AST using recursive-descent (precedence climbing). Reports parse errors and continues via panic-mode synchronisation (skips to next statement boundary).
  3. Interpreter — tree-walks the AST. Environments form a linked scope chain. Control flow uses C++ exceptions: ReturnException, BreakException, ContinueException.

Memory management

EZ uses std::shared_ptr for all heap-allocated values (strings, arrays, dictionaries, functions, instances). The GarbageCollector class counts allocations and triggers a "collection" every 1000 allocations — in this implementation collection is a no-op since shared_ptr handles reclamation automatically. The GC infrastructure is in place for a future mark-and-sweep upgrade.

Environment model

Each scope is an Environment object containing an unordered_map of variable names to Values, plus a shared pointer to the parent environment. Lookups walk the parent chain. Reads use shared_lock; writes use unique_lock — safe for multi-threaded spawn scenarios.

Value representation

Every EZ value is a Value struct holding a std::variant of twelve possible types: nullptr_t (nil), bool, double, StringPtr, ArrayPtr, FunctionPtr, NativeFnPtr, ClassPtr, InstancePtr, DictionaryPtr, FuturePtr, SuperPtr. Copies are cheap for scalars and shared-pointer increments for heap types.
