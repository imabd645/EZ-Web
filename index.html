Keywords that speak to you

EZ replaces cryptic programmer shorthand with plain English. Loops, conditionals, functions, and returns all map to words you already know.

Under the hood it's a full tree-walk interpreter with TCO, closures, a mark-and-sweep GC, and real OS threading — zero compromises on capability.

when → if other → else task → function give → return out → print repeat…to → for in → for-each escape → break skip → continue model → class

// What EZ brings

A complete language ecosystem

⚙️

Tree-Walk Interpreter

Built in C++17 with a recursive-descent parser, AST node visitor, and a 50,000-threshold mark-and-sweep garbage collector.

🔄

Tail Call Optimization

Parser marks tail-position calls; interpreter uses an exception-based trampoline so deeply recursive EZ programs never blow the stack.

⚡

Async / Await

Async functions spawn real OS threads via std::async, returning futures. await blocks until resolved. spawn() is fire-and-forget.

🏛️

Full OOP System

Classes (model), single inheritance (extends), super, self, visibility (hidden/shown), static members, and interfaces.

🔌

Native FFI

Load any DLL, resolve function pointers, and call into system APIs directly from EZ. Full raw memory read/write with pointer arithmetic.

🖼️

GUI Framework v6

50+ WinAPI wrappers: windows, buttons, inputs, listviews, tabs, timers, drag-and-drop, color picker, canvas drawing, Windows 11 dark mode.

📦

Built-in Package Manager

Install, list, and init packages directly via ez install. Packages live on GitHub under the EZ prefix. JSON manifest format.

🔐

Concurrency Primitives

GC-managed mutex objects, lock()/unlock(), atomic_add/atomic_inc/atomic_dec for lock-free counters.

🧠

Metaprogramming

getattr, setattr, hasattr, dynamic eval(), and type()/typeOf() for runtime reflection and code generation.

See it in action

// Object-Oriented Programming

model Animal {
    init(name, species) {
        self.name    = name
        self.species = species
    }
    task speak() {
        out self.name + " says hello!"
    }
}

model Dog extends Animal {
    init(name) {
        super(name, "Canis familiaris")
        self.tricks = []
    }
    task learn(trick) {
        push(self.tricks, trick)
        out self.name + " learned: " + trick
    }
    task speak() {
        out self.name + " says: Woof!"
    }
}

rex = new Dog("Rex")
rex.speak()
rex.learn("sit")
rex.learn("roll over")
out "Tricks: " + str(rex.tricks)
// Async / Await — real OS threads

async task fetchUser(id) {
    url  = "https://api.example.com/users/" + str(id)
    data = http_get(url)
    give parse_json(data)
}

async task fetchAll(ids) {
    futures = []
    repeat id in ids {
        push(futures, fetchUser(id))
    }
    results = []
    repeat f in futures {
        push(results, await f)
    }
    give results
}

users = await fetchAll([1, 2, 3, 4])
out "Fetched " + str(len(users)) + " users"
// Exception handling with typed errors

task divide(a, b) {
    when b == 0 {
        throw new ValueError("Division by zero")
    }
    give a / b
}

try {
    result = divide(10, 0)
    out result
} catch (e) {
    out "Caught: " + e.message
    // Full stack trace is available
}

// Custom error types via inheritance
model AuthError extends Error {
    init(msg, code) {
        self.message = msg
        self.code    = code
    }
}

throw new AuthError("Forbidden", 403)

// Native FFI — call Windows APIs directly

// Load Kernel32 dynamically
kernel32 = os_load_lib("kernel32.dll")
beep     = os_get_func(kernel32, "Beep")

// Call Beep(freq, duration)
os_call(beep, [800, 300])

// Allocate raw memory and write to it
ptr = os_alloc(64)
os_write_string(ptr, "Hello from EZ memory!")
val = os_read_string_ptr(ptr)
out val
os_free(ptr)

// Use WinAPI MessageBox
user32 = os_load_lib("user32.dll")
msgbox = os_get_func(user32, "MessageBoxA")
os_call(msgbox, [0, "Hello!", "EZ FFI", 0])

// Standard Library

Batteries included

EZ keeps the binary lean. Extended capabilities are modular packages installed via the built-in package manager.

🖥️ os.ez ez install os

Access environment variables, clipboard read/write, and process management. Implemented in pure EZ via native FFI over Kernel32.

env vars clipboard processes
📁 fs.ez ez install fs

Hardened filesystem operations using Kernel32 — list directories, check existence, get file size, copy, move, and delete with proper error handling.

listDir exists size copy/move
🌐 serve.ez ez install serve

DDoS-resistant, Express-style HTTP server running on native WS2_32 TCP sockets. Supports GET, POST, middleware, and JSON responses.

HTTP routing middleware native TCP
🔐 crypto.ez ez install crypto

Cryptography primitives in pure EZ — Base64 encode/decode, RC4 symmetric cipher, and hash utilities for authentication workflows.

base64 RC4 hashing
🔔 notify.ez ez install notify

Windows system tray balloon notifications. Supports info, warning, and error icon types with custom title and message text.

tray balloon tips icons
🗄️ SQLite ORM built-in FFI

Embedded SQLite3 database access via the FFI layer. Perform queries, parameterized statements, and schema migrations from EZ scripts.

SQLite3 queries params
