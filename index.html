Section 01 — Overview

The EZ Language

EZ is a modern, dynamically-typed interpreted programming language designed for clarity and expressive power. Its syntax is deliberately English-like, favouring readable keywords over cryptic punctuation. EZ is compiled to an AST and executed by a tree-walking interpreter written in C++.

Core Design Principles

  • Readable syntax — keywords like when, other, repeat, task, give, escape, and get map directly to natural-language intent.
  • Dynamic typing — no type annotations needed. Values carry their own type at runtime.
  • Garbage collected — a mark-and-sweep GC handles memory automatically, with cycle detection via a doubly-linked intrusive object list.
  • Full OOP — first-class classes (model), single inheritance, interfaces, structs, and operator overloading.
  • Async-first — built-in async/await, spawn (threads), and mutex primitives.
  • Rich standard library — math, string, I/O, networking (HTTP via libcurl), PDF generation, buffers, regex, and concurrency utilities are all built in.
  • Tail call optimisation — recursive functions are safe from stack overflow when calls are in tail position.
  • Package manager — install, init, and list packages from GitHub repositories.

Architecture at a Glance

EZ source code travels through three pipeline stages:

Source (.ez file)
   │
   ▼
[Lexer]  → Tokenises source into a flat token stream (Token.h)
   │
   ▼
[Parser] → Builds an Abstract Syntax Tree — ExprPtr / StmtPtr nodes (AST.h)
   │
   ▼
[Interpreter] → Tree-walks the AST, maintaining an Environment chain
               │
               ├─ Evaluates expressions → Value (Value.h)
               ├─ Executes statements
               └─ Calls built-in / user-defined functions

File Extension

EZ source files use the .ez extension. Example: hello.ez, server.ez, app.ez.

Tip: EZ is case-sensitive. All built-in keywords are lowercase (when, not When). Identifiers can use camelCase or snake_case — both are valid.
Section 02 — Setup

Installation & Usage

Running the Interpreter

The EZ interpreter is a compiled C++ binary named ez (or ez.exe on Windows). Invoke it from the command line:

# Run a script file
ez myprogram.ez

# Launch the interactive REPL
ez

# Show help
ez --help

CLI Reference

CommandDescription
ezLaunch the REPL (interactive mode). Type exit to quit.
ez <file.ez>Execute a script file. Exit codes: 65 = parse error, 70 = runtime error.
ez install <pkg> [ver]Install a package from the ezlib GitHub repository.
ez listList all currently installed packages.
ez init <name>Scaffold a new EZ package with package.ez and main.ez.
ez --helpDisplay syntax reference.

REPL (Interactive Mode)

Running ez with no arguments starts the Read-Eval-Print Loop. The REPL supports multi-line input — it waits for you to close all open braces {} before evaluating. Use ... as the continuation prompt.

EZ Language Interpreter v1.0
Type 'exit' to quit

>>> x = 10
>>> out x * 2
20
>>> task greet(name) {
...   out "Hello, " + name
... }
>>> greet("World")
Hello, World

Build Dependencies

  • C++17 compiler (MSVC, GCC, Clang)
  • libcurl — for HTTP networking built-ins
  • Windows SDK — for GUI, console colour, and FFI features (optional on other platforms)

Package Storage

Packages are installed to C:/ezlib/<package-name>/ by default. A packages.json manifest tracks installed packages and their versions.

Truthiness Rules

In EZ, the truthiness of a value is simple:

  • Falsy: nil and false
  • Truthy: everything else — including 0, empty strings "", and empty arrays []
when 0    { out "truthy" }  // prints "truthy" ← 0 is truthy in EZ!
when ""   { out "truthy" }  // prints "truthy"
when nil  { out "truthy" }  // NOT printed — nil is falsy
when false{ out "truthy" }  // NOT printed — false is falsy

Type Checking at Runtime

x = 42
out type(x)       // "integer"
out typeOf("hi")  // "string"
out type([1,2])   // "array"
Section 06 — Variables

Variables & Assignment

EZ uses dynamic typing with implicit declaration — you create a variable simply by assigning to it. There is no var, let, or const keyword.

Basic Assignment

// Create and assign in one step
name   = "Alice"
age    = 25
score  = 98.5
active = true
data   = nil

// Reassign to any type at any time
age = "twenty-five"  // now a string

Compound Assignment Operators

x = 10
x += 5   // x = 15
x -= 3   // x = 12
x *= 2   // x = 24
x /= 4   // x = 6

Static Variables

The static keyword inside a task creates a variable that persists across calls to that function — it is initialised only once:

task counter() {
    static count = 0
    count += 1
    out count
}

counter()  // 1
counter()  // 2
counter()  // 3

Scope Rules

EZ uses lexical (static) scoping with an environment chain. Each block {} creates a new child scope. Variable lookup walks the parent chain. Assignment to an existing variable updates it in the nearest enclosing scope where it was defined.

x = "global"

task outer() {
    x = "outer"      // updates global x
    y = "local"      // new in outer scope
    
    task inner() {
        out x          // "outer" — closes over x
        out y          // "local"
    }
    inner()
}
outer()
Warning: If you assign to a name that was never defined anywhere in the scope chain, EZ will auto-define it in the current scope (like a global). Be careful to avoid accidental globals inside tasks.

Section 08 — Control Flow

when / other (If-Else)

EZ uses when instead of if and other instead of else. The syntax is identical in structure to C-style if/else.

Basic Syntax

when condition {
    // executes if condition is truthy
}

when / other

x = 15

when x > 10 {
    out "big"
} other {
    out "small"
}

Chained Conditions (else-if)

score = 75

when score >= 90 {
    out "A"
} other when score >= 80 {
    out "B"
} other when score >= 70 {
    out "C"
} other {
    out "F"
}

Inline (Single Statement) when

when x > 0 { out "positive" }

Complex Conditions

age = 25
hasLicense = true

when age >= 18 and hasLicense {
    out "Can drive"
}

when age < 16 or not hasLicense {
    out "Cannot drive"
}
Tip: The condition of a when does not need parentheses. Both when x > 0 { and when (x > 0) { are valid — the parentheses are optional.
Section 09 — Loops

while Loop

The while loop executes its body as long as the condition remains truthy.

Basic while

i = 0
while i < 5 {
    out i
    i += 1
}

Infinite Loop with escape

count = 0
while true {
    count += 1
    when count >= 10 {
        escape   // break
    }
}
out count  // 10

While with skip (continue)

i = 0
while i < 10 {
    i += 1
    when i % 2 == 0 { skip }  // skip evens
    out i   // prints 1 3 5 7 9
}

Do-While Pattern

EZ has no do-while keyword, but you can emulate it:

firstRun = true
while firstRun or condition {
    firstRun = false
    // body executes at least once
}
Section 10 — Loops

repeat Loop (For Loop)

The repeat statement is EZ's for-loop. It iterates a variable from a start value to an end value (exclusive upper bound).

Basic repeat

repeat i=0 to 5 {
    out i
}
// prints 0, 1, 2, 3, 4

Reverse / Custom Start

// Count from 3 to 9
repeat i=3 to 10 {
    out i
}

Nested repeat

repeat row=0 to 3 {
    repeat col=0 to 3 {
        out row + "," + col
    }
}

Using repeat with escape and skip

repeat i=0 to 100 {
    when i == 50 { escape }   // stop at 50
    when i % 3 == 0 { skip } // skip multiples of 3
    out i
}
Note: The repeat variable (i above) is scoped to the loop body. The end value is computed once at the start of the loop and is exclusive — the loop runs while i < end.

Iterating Over a Range Array

// range(n) returns [0,1,...,n-1]
// range(start, end) returns [start,...,end-1]
nums = range(5)
get n in nums {
    out n
}
Section 11 — Loops

get…in (Foreach Loop)

The get…in construct iterates over every element of an array. It's EZ's equivalent of a foreach or for-of loop.

Basic get…in

fruits = ["apple", "banana", "cherry"]

get fruit in fruits {
    out fruit
}

Iterating Numbers

total = 0
scores = [88, 92, 75, 95]

get s in scores {
    total += s
}
out total / len(scores)  // average

Nested get…in

matrix = [[1,2],[3,4],[5,6]]

get row in matrix {
    get cell in row {
        out cell
    }
}

With escape and skip

items = ["a", "b", "stop", "c"]

get item in items {
    when item == "stop" { escape }
    out item
}
// prints "a" "b"

Iterating Dictionary Keys

person = {"name": "Alice", "age": 30}

get k in keys(person) {
    out k + ": " + person[k]
}
Section 12 — Control Flow

escape & skip

escape (break)

escape immediately exits the innermost loop. It is EZ's break.

repeat i=0 to 100 {
    when i == 5 {
        escape   // exit the loop
    }
    out i
}
// prints 0 1 2 3 4

skip (continue)

skip jumps immediately to the next iteration of the innermost loop, skipping the remaining body. It is EZ's continue.

repeat i=0 to 10 {
    when i % 2 == 0 {
        skip   // skip even numbers
    }
    out i
}
// prints 1 3 5 7 9

Nested Loop Control

Both escape and skip always apply to the innermost enclosing loop. To break an outer loop from inside a nested loop, use a flag variable:

found = false
repeat i=0 to 10 {
    when found { escape }
    repeat j=0 to 10 {
        when i * j == 42 {
            found = true
            escape
        }
    }
}
Section 13 — Functions

task (Functions)

Functions in EZ are defined with the task keyword. They are first-class values — you can pass them as arguments, assign them to variables, and return them from other tasks.

Basic task Definition

task greet(name) {
    out "Hello, " + name + "!"
}

greet("Alice")  // Hello, Alice!

Multi-parameter task

task add(a, b) {
    give a + b
}

result = add(3, 4)
out result   // 7

Tasks are First-Class Values

task double(x) { give x * 2 }

fn = double       // assign function to variable
out fn(5)        // 10

task apply(func, val) {
    give func(val)
}
out apply(double, 7)  // 14

Recursive Tasks

task factorial(n) {
    when n <= 1 { give 1 }
    give n * factorial(n - 1)
}

out factorial(10)  // 3628800

Tail Call Optimisation

EZ supports tail call optimisation. When a recursive call is the last operation before give, the interpreter reuses the current stack frame, preventing stack overflow for deep recursion:

task sum(n, acc) {
    when n == 0 { give acc }
    give sum(n - 1, acc + n)  // tail call
}
out sum(100000, 0)  // no stack overflow
Call Stack: EZ enforces a maximum call depth of 500 frames (MAX_CALL_DEPTH). For deep recursion without TCO, use an iterative approach or ensure your recursion is in tail position.
Section 14 — Functions

give (Return)

give is EZ's return statement. It exits the current task and optionally provides a value back to the caller.

Returning a Value

task square(x) {
    give x * x
}

out square(5)   // 25

Early Return

task safeDivide(a, b) {
    when b == 0 {
        give nil   // early return
    }
    give a / b
}

result = safeDivide(10, 0)
out result   // nil

Bare give

A give with no value returns nil:

task doSomething() {
    when badCondition { give }  // returns nil
    // ... more work ...
}

Returning Multiple Values (Array)

task minMax(arr) {
    mn = arr[0]
    mx = arr[0]
    get v in arr {
        when v < mn { mn = v }
        when v > mx { mx = v }
    }
    give [mn, mx]
}

result = minMax([3, 1, 7, 2, 9])
out result[0]  // 1
out result[1]  // 9
Section 15 — Functions

Lambdas & Closures

EZ supports anonymous functions (lambdas) using the arrow syntax. They capture variables from the enclosing scope (closures).

Arrow Lambda Syntax

// Single expression — implicitly returned
square = (x) => x * x

// Multi-statement body
abs_val = (x) => {
    when x < 0 { give -x }
    give x
}

out square(6)    // 36
out abs_val(-5) // 5

Lambdas as Arguments

nums = [1, 2, 3, 4, 5]

// map — apply function to every element
doubled = map(nums, (x) => x * 2)
out doubled   // [2, 4, 6, 8, 10]

// filter — keep elements matching predicate
evens = filter(nums, (x) => x % 2 == 0)
out evens     // [2, 4]

// reduce — fold to single value
total = reduce(nums, (acc, x) => acc + x, 0)
out total     // 15

Closures

A lambda closes over variables from the surrounding scope. The captured variables are "remembered" even after the outer scope has returned:

task makeCounter(start) {
    count = start
    give () => {
        count += 1
        give count
    }
}

counter = makeCounter(10)
out counter()   // 11
out counter()   // 12
out counter()   // 13

Higher-Order Functions

task compose(f, g) {
    give (x) => f(g(x))
}

addOne  = (x) => x + 1
double  = (x) => x * 2

addOneThenDouble = compose(double, addOne)
out addOneThenDouble(3)  // (3+1)*2 = 8
Section 16 — Functions

Variadic Functions & Default Parameters

Default Parameter Values

Parameters can have default values, used when the caller omits that argument:

task greet(name, greeting="Hello") {
    out greeting + ", " + name + "!"
}

greet("Alice")             // Hello, Alice!
greet("Bob", "Hi")        // Hi, Bob!

Variadic Functions

Add ... after the last parameter name to collect remaining arguments into an array:

task sum(args...) {
    total = 0
    get n in args {
        total += n
    }
    give total
}

out sum(1, 2, 3)        // 6
out sum(10, 20, 30, 40)  // 100

Spread Operator

The ... spread operator unpacks an array into individual arguments when calling a function:

nums = [1, 2, 3]

task add3(a, b, c) { give a + b + c }

out add3(...nums)   // 6 — spreads the array
Tip: Default values can be any expression, evaluated at call time — not at definition time. So you can use other parameters in a default value expression.
Section 17 — OOP

model (Classes)

EZ uses the model keyword to define classes. A model bundles data (properties) and behaviour (methods). Instances are created with new.

Defining a Model

model Animal {
    init(name, sound) {
        self.name  = name
        self.sound = sound
    }
    
    task speak() {
        out self.name + " says " + self.sound
    }
    
    task getName() {
        give self.name
    }
}

Creating Instances with new

dog = new Animal("Dog", "Woof")
cat = new Animal("Cat", "Meow")

dog.speak()              // Dog says Woof
cat.speak()              // Cat says Meow
out dog.getName()       // Dog
out dog.name            // access property directly

Access Modifiers: shown / hidden

Members can be marked shown (public) or hidden (private). Private members are only accessible from within the model itself:

model BankAccount {
    init(balance) {
        hidden self._balance = balance
    }
    
    shown task deposit(amount) {
        self._balance += amount
    }
    
    shown task getBalance() {
        give self._balance
    }
    
    hidden task _validate(amount) {
        give amount > 0
    }
}

acc = new BankAccount(1000)
acc.deposit(500)
out acc.getBalance()   // 1500

self Reference

self refers to the current instance inside any method or init block. It is equivalent to this in other languages.

toString Override

model Point {
    init(x, y) {
        self.x = x
        self.y = y
    }
    task toString() {
        give "(" + self.x + "," + self.y + ")"
    }
}

p = new Point(3, 4)
out p    // (3,4) — uses toString
Section 18 — OOP

Inheritance & extends

EZ supports single-class inheritance. A child model inherits all methods from its parent and can override them. Use super to call the parent's methods.

Basic Inheritance

model Shape {
    init(color) {
        self.color = color
    }
    task describe() {
        out "A " + self.color + " shape"
    }
    task area() {
        give 0
    }
}

model Circle extends Shape {
    init(color, radius) {
        super.init(color)       // call parent constructor
        self.radius = radius
    }
    task area() {             // override parent method
        give 3.14159 * self.radius * self.radius
    }
    task describe() {
        super.describe()        // call parent method
        out "  Radius: " + self.radius
    }
}

c = new Circle("red", 5)
c.describe()
out c.area()    // 78.53975

Method Overriding

A child model can redefine any method from its parent. The parent's version is still accessible via super.methodName().

Multi-Level Inheritance

model Vehicle { ... }
model Car extends Vehicle { ... }
model ElectricCar extends Car { ... }
Note: EZ supports only single inheritance (one parent). For multiple-interface contracts, use implements (see next section).
Section 19 — OOP

interface & implements

Interfaces define a contract — a list of method names that any implementing model must provide. This enables polymorphism and type-checked duck typing.

Defining an Interface

interface Drawable {
    draw
    resize
}

interface Serializable {
    serialize
    deserialize
}

Implementing an Interface

model Rectangle implements Drawable {
    init(w, h) {
        self.w = w
        self.h = h
    }
    
    task draw() {
        out "Drawing rect " + self.w + "x" + self.h
    }
    
    task resize(factor) {
        self.w *= factor
        self.h *= factor
    }
}

r = new Rectangle(10, 20)
r.draw()             // Drawing rect 10x20
r.resize(2)
r.draw()             // Drawing rect 20x40

Multiple Interfaces

model Document implements Drawable, Serializable {
    init(content) { self.content = content }
    task draw()        { out self.content }
    task resize(f)     { }
    task serialize()   { give self.content }
    task deserialize(s){ self.content = s }
}
Note: The interpreter verifies at runtime that implementing models define all required interface methods. Missing methods cause a runtime error.
Section 20 — OOP

struct

A struct is a lightweight, named data container with named fields. Unlike a model, structs have no custom init or methods — they're pure data records.

Defining a Struct

struct Point { x, y }
struct Color { r, g, b }
struct Rect  { x, y, width, height }

Creating Struct Instances

Use new to create a struct instance. Arguments are mapped positionally to the fields in declaration order:

p = new Point(10, 20)
out p.x   // 10
out p.y   // 20

bg = new Color(255, 128, 0)
out bg.r  // 255

Modifying Struct Fields

p = new Point(0, 0)
p.x = 100
p.y = 200
out p.x   // 100

Section 21 — OOP

static Members

Static members belong to the class itself, not to any particular instance. They are shared across all instances and can be accessed directly on the model name.

Static Properties & Methods

model Counter {
    static count = 0
    
    init() {
        Counter.count += 1
        self.id = Counter.count
    }
    
    static task getCount() {
        give Counter.count
    }
    
    static task reset() {
        Counter.count = 0
    }
}

a = new Counter()
b = new Counter()
c = new Counter()

out Counter.getCount()   // 3
out Counter.count        // 3
Counter.reset()
out Counter.count        // 0

Static Variables in Tasks (Function-level)

Inside a regular task, the static keyword creates a persistent variable scoped to that function — initialised only on the first call:

task uniqueId() {
    static nextId = 1
    id = nextId
    nextId += 1
    give id
}

out uniqueId()   // 1
out uniqueId()   // 2

Section 23 — Async

async / await

EZ has native async/await support. An async task or expression returns a Future — a handle to a value that will be available later. await blocks until the future resolves.

Defining an async task

async task fetchData(url) {
    response = await http_get(url)
    give response
}

// Call it and await the result
data = await fetchData("https://api.example.com/data")
out data

await Expression

// fetch() returns a Future
fut = fetch("https://httpbin.org/get")

// await resolves it to the value
result = await fut
out result

Parallel Async Operations

// Fire two requests in parallel
f1 = fetch("https://api.example.com/users")
f2 = fetch("https://api.example.com/posts")

// Await both
users = await f1
posts = await f2
out users
out posts

sync() — Alias for await

The built-in sync(future) is identical to await future and is useful when calling from non-async contexts:

result = sync(fut)   // same as await fut
Under the hood: Futures in EZ are backed by std::shared_future<Value> from the C++ standard library, running on real OS threads.
Section 24 — Concurrency

spawn & Futures

spawn runs a function on a new OS thread and immediately returns a Future. This is the primary way to achieve true parallelism in EZ.

Basic spawn

    total = 0
    repeat i=0 to n {
        total += i
    }
    give total
}

// Block until result is ready

out "Result: " + result

    stop(1000)   // sleep 1 second

    f = spawn((id) => {
        stop(id * 100)
        give "Thread " + id + " done"

}

// Await all results

    out await f
}

Section 40 — Internals

Garbage Collector

EZ uses a mark-and-sweep garbage collector with cycle detection. It automatically frees memory for arrays, dictionaries, functions, classes, instances, buffers, and mutexes.

How It Works

All heap-allocated objects (GCObject subclasses) are registered in a global doubly-linked intrusive list. The GC runs in three phases:

  1. Reset Marks — Traverse the entire object list and clear all gc_marked flags.
  2. Mark Phase — Starting from roots (global environment, call stack, all active interpreter environments, and temporary roots), recursively mark every reachable object.
  3. Sweep Phase — Any object not marked is unreachable. Call gc_clear() on it (breaks reference cycles) and allow it to be freed.

GC Roots

  • The global Environment
  • Every active Interpreter's current environment and environment stack
  • Explicit temporary roots (used during native calls)

Allocation Threshold

The GC runs automatically when the number of allocations since the last collection exceeds the threshold (default: 50,000 allocations). You can observe GC statistics but cannot configure the threshold from EZ script directly.

Thread Safety

The GC uses multiple mutexes: listMutex for the object list, interpretersMutex for the interpreter registry, tempRootsMutex for temporary roots, and collectMutex to prevent concurrent collections.

Types Tracked by GC

EZArray
Marks all elements
EZDictionary
Marks all values
EZInstance
Marks class + properties
EZFunction
Marks closure + static env
EZClass
Marks parent + methods
Environment
Marks parent + variables
EZBuffer
Marks self only
EZMutex
Marks self only

Expression Evaluation Flow

Interpreter::evaluate(ExprPtr)
  │
  ├─ LiteralExpr    → return Value directly
  ├─ IdentifierExpr → Environment::get(name) → Value
  ├─ BinaryExpr     → evaluate left + right → apply op
  │                   → check for operator overload (__add__ etc.)
  ├─ CallExpr       → evaluate callee + args → callFunction()
  │   ├─ NativeFunction  → call C++ lambda
  │   ├─ EZFunction      → push frame, execute body, pop frame
  │   └─ EZClass         → create EZInstance, run init()
  ├─ IndexExpr      → evaluate object + index → array/dict lookup
  ├─ AssignExpr     → evaluate value → Environment::assign()
  ├─ PropertyAccess → EZInstance::properties[name]
  ├─ AwaitExpr      → future->get() (blocks thread)
  └─ AsyncExpr      → wrap in std::async → return Future Value

Control Flow Exceptions

EZ implements control flow using C++ exceptions internally for efficiency. This avoids threading explicit return/break signals through the interpreter:

  • ReturnException — carries the return value from give
  • BreakException — signals escape
  • ContinueException — signals skip
  • TailCallException — carries callee + args for TCO trampoline
  • RuntimeError — carries message + line for user errors

Environment Chain

Scopes form a parent chain. The global environment has no parent. Each block creates a child environment. getPtr() returns a direct pointer for in-place modification (used for index assignment). The mutex is a shared_mutex — reads are shared-locked, writes are exclusive-locked.

Extending EZ: To add a new built-in function, register it with interp.defineGlobal("name", Value::makeNativeFunction("name", arity, lambda)) in any of the Builtins_*.cpp files and add its declaration to registerBuiltins() in Builtins.cpp.
